# 🎙️ Emotion Recognition from Speech

Recognizing human emotions — happy, sad, angry, neutral, calm, fearful, disgust, and surprised — from raw speech audio using deep learning and acoustic signal processing, with a CNN-BiLSTM classifier and an interactive Streamlit demo.

## 📌 Objective

Classify the emotional state of a speaker directly from a short audio clip, without relying on transcripts or text — purely from acoustic/prosodic cues in the voice.

## 🧠 Approach

- **Signal processing:** extract MFCCs, delta-MFCCs, Chroma, log-Mel-spectrogram, Zero-Crossing-Rate, and RMS energy from each clip.
- **Data augmentation:** noise injection, pitch shifting, and time stretching to improve generalization across speakers and recording conditions.
- **Model:** a CNN + Bidirectional LSTM (CNN-BiLSTM) architecture — convolutional layers learn local spectral patterns, BiLSTM layers model how those patterns evolve over time.
- **Evaluation:** accuracy, per-class classification report, and confusion matrix on a held-out test set.

## 📦 Supported Datasets

| Dataset | Description | Source |
|---|---|---|
| **RAVDESS** | Ryerson Audio-Visual Database of Emotional Speech and Song — 24 actors, 8 emotions | [Zenodo](https://zenodo.org/record/1188976) / Kaggle: `uwrfkaggler/ravdess-emotional-speech-audio` |
| **TESS** | Toronto Emotional Speech Set — 2 actresses, 7 emotions | Kaggle: `ejlok1/toronto-emotional-speech-set-tess` |
| **EMO-DB** | Berlin Database of Emotional Speech (German) — 10 actors, 7 emotions | [emodb.bilderbar.info](http://emodb.bilderbar.info/) / Kaggle: `piyushagni5/berlin-database-of-emotional-speech-emodb` |

The notebook auto-detects whichever datasets are present and parses each one's filename convention automatically, so you can train on one dataset or combine all three.

## 🧪 Training Pipeline (Notebook)

1. **Download a dataset** (RAVDESS / TESS / EMO-DB) and point `DATASET_PATHS` in the Configuration cell to the correct folder(s).
2. **Run all cells** — the notebook will:
   - parse filenames into a unified 8-emotion label set,
   - load and pad/truncate each clip to a fixed length,
   - extract MFCC + delta-MFCC + Chroma + log-Mel + ZCR + RMS features,
   - optionally augment the data (noise/pitch/time-stretch),
   - split into stratified train/validation/test sets,
   - standardize features with `StandardScaler` (fit on train only),
   - train the CNN-BiLSTM model with early stopping and LR scheduling,
   - evaluate on the test set (accuracy, classification report, confusion matrix),
   - save the trained model, scaler, and label encoder to disk.

### Model Architecture

```
Input (time_steps, n_features)
  → Conv1D(128) → BatchNorm → MaxPool → Dropout(0.3)
  → Conv1D(256) → BatchNorm → MaxPool → Dropout(0.3)
  → Bidirectional LSTM(128, return_sequences=True) → Dropout(0.3)
  → Bidirectional LSTM(64) → Dropout(0.3)
  → Dense(128, relu, L2 reg) → Dropout(0.4)
  → Dense(n_classes, softmax)
```

## ▶️ Running the Demo App

Once the notebook has saved `emotion_recognition_model.keras`, `feature_scaler.pkl`, and `label_encoder.pkl` into the project folder:

```bash
streamlit run emotion_recognition_app.py
```

Open the local URL Streamlit prints, upload a short speech clip (≈2–6 seconds), and the app will display:

- the waveform and MFCC heatmap of the clip,
- the predicted emotion with confidence,
- a probability breakdown across all 8 emotion classes,
- a "Pipeline Details" tab summarizing the exact feature/model configuration used.
